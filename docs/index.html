Keysto foi ajustado e documentado.

Esta pagina resume o que foi lido, auditado, corrigido, empacotado e publicado hoje no projeto Keystone BTC Proof. O objetivo foi sair de uma biblioteca solta e entregar algo executavel, auditado, acessivel via navegador e publicavel por link.

Visao geral

O que foi entregue

O projeto deixou de ser apenas uma crate Rust fechada e passou a ter uma forma clara de uso. Foi finalizada a camada de verificacao, criada uma interface web, preparado um frontend estatico para GitHub Pages e organizada uma publicacao publica por link.

Tambem foi feita uma revisao de logica para fechar buracos de rede, janela temporal, altura maxima de inclusao e confianca do tip. Onde havia risco material, o codigo foi corrigido e coberto por teste.

Estado atual

Como o projeto ficou

  • Biblioteca de verificacao com fluxo de avaliacao fechado.
  • Binario com comandos `template`, `verify` e `serve`.
  • Interface navegavel separada em `app.html`.
  • Pagina principal transformada em documento de entrega.
  • Publicacao automatica no GitHub Pages via GitHub Actions.
  • Repositorio remoto publicado em `main`.
Linha do tempo

Ajustes realizados hoje

Auditoria do codigo Foi feita leitura completa da base Rust para entender o fluxo de `ClaimBundle`, headers, Merkle proof, Taproot, ordem e settlement.
Correcao da adjudicacao Foram fechados problemas em rede Bitcoin esperada, altura maxima de inclusao, validacao do tip aceito e checagens de contexto L2.
Empacotamento executavel Foi criado um binario consumivel, com JSON de exemplo, verificacao por arquivo ou stdin e modo servidor HTTP para uso em navegador.
Publicacao web Foi preparada uma camada estatica para GitHub Pages e separado o frontend em duas experiencias: documento de entrega e aplicacao interativa.
Exposicao publica O repositorio foi publicado no GitHub, o workflow de Pages foi habilitado e o link publico foi validado.
Entregaveis tecnicos

Arquivos e superficies principais

Backend Rust

`src/main.rs` expande a crate com CLI e servidor HTTP, enquanto `src/evaluate.rs` concentra a avaliacao do bundle.

Frontend estatico

`docs/index.html` virou o documento publico. `docs/app.html` guarda a ferramenta interativa para testes no navegador.

Deploy

O workflow `pages.yml` publica automaticamente o conteudo de `docs/` no GitHub Pages.

Operacao

Scripts em `service/` continuam disponiveis para rodar a interface web na VPS e manter tunel/servico.
